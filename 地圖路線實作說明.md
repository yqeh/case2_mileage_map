# 地圖路線實作說明

## 📍 系統使用兩種方式生成地圖路線

### 方式一：簡單直線連接（單筆計算使用）

**使用場景：** `/api/calculate/distance` 單筆計算

**實作方法：** `download_static_map()`

**流程：**
1. 使用 **Geocoding API** 將地址轉換為經緯度座標
2. 使用 **Static Maps API** 建立地圖 URL：
   ```
   https://maps.googleapis.com/maps/api/staticmap?
   size=800x600&
   markers=color:red|label:S|{起點緯度},{起點經度}&
   markers=color:green|label:E|{終點緯度},{終點經度}&
   path=color:0x0000ff|weight:5|{起點緯度},{起點經度}|{終點緯度},{終點經度}&
   key={API_KEY}
   ```
3. 下載地圖圖片並儲存

**特點：**
- ✅ 速度快
- ❌ 只顯示起點和終點的直線連接
- ❌ 不顯示實際道路路線

---

### 方式二：完整路線 Polyline（批次計算使用）⭐ **推薦**

**使用場景：** `/api/calculate/batch` 批次計算

**實作方法：** `get_route_detail()` + `download_static_map_with_polyline()`

**流程：**

#### 步驟 1：取得實際路線資訊
使用 **Directions API** 取得完整路線：
```python
directions_result = self.gmaps.directions(
    origin_address,
    dest_address,
    mode='driving',
    language='zh-TW'
)
```

**取得資料：**
- ✅ 實際道路路線的 **polyline**（路線編碼）
- ✅ 逐步導航指示（steps）
- ✅ 總距離和時間
- ✅ 路線詳細資訊

#### 步驟 2：使用 Polyline 生成地圖
使用 **Static Maps API** 配合 polyline 顯示完整路線：
```
https://maps.googleapis.com/maps/api/staticmap?
size=800x600&
path=enc:{polyline編碼}&
key={API_KEY}
```

**特點：**
- ✅ 顯示**實際道路路線**（不是直線）
- ✅ 路線會沿著真實道路繪製
- ✅ 可以在地圖上顯示公里數
- ✅ 包含完整的導航步驟資訊

---

## 🔧 技術細節

### Polyline 是什麼？

**Polyline** 是 Google Maps 用來編碼路線的格式，它將複雜的路線座標壓縮成一個字串。

**範例：**
```
原始路線：多個經緯度點
(25.0330, 121.5654) → (25.0340, 121.5664) → (25.0350, 121.5674) ...

編碼後：一個字串
"abcxyz123..." (polyline 編碼)
```

### 為什麼使用 Polyline？

1. **節省 API 請求大小**：不需要傳送大量座標點
2. **顯示真實路線**：路線會沿著實際道路繪製
3. **精確度高**：路線與 Google Maps 導航一致

---

## 📊 兩種方式對比

| 項目 | 方式一（直線） | 方式二（Polyline） |
|------|---------------|-------------------|
| **API 使用** | Geocoding + Static Maps | Directions + Static Maps |
| **路線顯示** | 起點到終點直線 | 實際道路路線 |
| **速度** | 快 | 較慢（需要計算路線） |
| **準確度** | 低（直線距離） | 高（實際道路距離） |
| **使用場景** | 單筆快速計算 | 批次完整計算 |
| **成本** | 較低 | 較高（Directions API 計費較高） |

## 🎯 目前系統實作

### 批次計算（主要功能）

```python
# 1. 取得詳細路線（包含 polyline）
route_detail = maps_service.get_route_detail(origin, destination)

# 2. 使用 polyline 生成地圖
map_image = maps_service.download_static_map_with_polyline(
    polyline=route_detail['polyline'],
    origin_address=origin,
    destination_address=destination,
    distance_km=route_detail['distance_km']  # 會顯示在地圖上
)
```

**結果：**
- ✅ 地圖顯示實際道路路線
- ✅ 地圖左上角顯示公里數
- ✅ 路線沿著真實道路繪製

## 📝 相關 API

### Google Maps Directions API
- **用途**：計算路線、取得導航指示
- **回傳**：路線資訊、polyline、距離、時間、步驟

### Google Maps Static Maps API
- **用途**：生成靜態地圖圖片
- **支援**：markers（標記）、path（路線）、polyline（編碼路線）

### Google Maps Geocoding API
- **用途**：地址轉換為經緯度
- **使用場景**：簡單地圖生成
